# 🛡️ 백엔드 운영 철칙 (Backend Operations Golden Rules)

이 워크플로우는 백엔드, 데이터베이스(Supabase), 태블릿(SSH) 작업을 수행할 때 AI 에이전트가 반드시 준수해야 하는 행동 지침입니다.

## 1. 🚫 복합 명령어 사용 금지 (No Composite Commands)

- 터미널이나 파워쉘에서 `&&`, `;`, `|` 등을 사용하여 명령어를 묶어서 보내지 마십시오.
- 반드시 **명령어 하나를 단독으로 전송**하십시오.

## 2. 🚦 에러 즉시 정지 (Stop on Error)

- 명령어 실행 후 `Exit Code 1` 또는 기타 에러 메시지가 발견되면, 다음 단계를 수행하지 마십시오.
- 즉시 멈추고 원인을 분석하여 사용자에게 보고한 뒤 대안을 마련하십시오.

## 3. 🔍 답변 전량 정독 (Read Every Line)

- 터미널 응답의 첫 줄부터 끝 줄까지 모두 정독하십시오.
- 특정 모듈이 없다거나(ModuleNotFound), 권한이 부족하다는(Permission denied) 사소한 경고도 놓치지 마십시오.

## 4. 📱 태블릿 원격 보안 (Tablet Remote Safety)

- SSH 연결 시 비밀번호(`aisapiens`) 입력 프롬프트를 항상 대기하십시오.
- 명령어가 태블릿 환경(Termux)에서 씹히지 않도록, 실행 결과를 눈으로 확인한 후 다음 명령으로 넘어가십시오.

## 5. 🏗️ DB 구조 변경 (DDL Rules)

- 인덱스 생성, 테이블 수정 등 DB 구조 변경은 가급적 **Supabase Dashboard SQL Editor** 사용을 권장하십시오.
- CLI 사용 시에는 반드시 연결 상태를 먼저 검증하십시오.
